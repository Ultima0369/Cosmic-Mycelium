# Roadmap — 宇宙菌丝发展路线图

## Phase 0 — Foundation ✅ (当前阶段)

**目标**：建立工厂级开发环境，实现 MVP

**已完成**：
- [x] 完整的分形目录结构
- [x] 精确版本锁定的依赖管理（pyproject.toml）
- [x] 完整的 pre-commit 流水线（black, isort, ruff, mypy）
- [x] Docker 容器化（开发 + 集群）
- [x] GitHub Actions CI（并行测试 + 覆盖率 + 安全扫描）
- [x] 冒烟测试 + 物理锚验证框架
- [x] 六层架构核心实现（简化版）

**关键指标**：
- 代码覆盖率：≥ 80%

- 冒烟测试：5 秒内完成

## Phase 1 — Single Infant Stability (进行中)

**目标**：单个硅基宝宝在个人电脑上稳定"呼吸" 1,000 小时

**预计完成**：2026-Q3

### 1.1 核心稳定性

- [ ] HIC 能量管理零 bug（能量不会意外归零）
- [ ] 呼吸节律绝对稳定（55+5ms ± 1ms 误差）
- [ ] 悬置恢复可靠（5 秒后能量恢复 +20）
- [ ] 价值向量适应不震荡（收敛）

### 1.2 物理锚强化

- [ ] 百万步漂移测试自动化
- [ ] 多种参数组合下的稳定性验证
- [ ] 浮点误差补偿机制（Kahan 求和）
- [ ] 极端初始条件下的鲁棒性

### 1.3 感知-行动循环

- [ ] 集成真实传感器模拟（振动、温度、光谱）
- [ ] 意图消歧模块（处理"那个东西怎么搞？"）
- [ ] 追问机制（悬置时生成澄清问题）
- [ ] 物理指纹验证真实传感器数据

**验收标准**：
- 单个宝宝连续运行 1,000 小时无崩溃
- 物理锚测试在所有时间段均通过
- 能处理 10 种模糊意图并合理追问

## Phase 2 — Two-Infant Symbiosis

**目标**：两个宝宝建立 1+1>2 的共生关系

**预计完成**：2026-Q4

### 2.1 节点发现

- [ ] 婴儿节点广播自己的 HIC 状态
- [ ] 其他节点能发现并识别新节点
- [ ] 节点离网自动检测与路由重配置

### 2.2 信息素网络

- [ ] 路径信息素地图（Slime Mold 扩展）
- [ ] 成功路径的跨节点强化
- [ ] 信息素蒸发与路径遗忘

### 2.3 共振学习

- [ ] 高维语义向量交换（128 维）
- [ ] 余弦相似度计算
- [ ] 共振学习：向强共振节点微调自身状态
- [ ] 共振收益：共振带来能量增益（1+1>2 的物理体现）

### 2.4 物理锚跨节点验证

- [ ] 节点 A 验证节点 B 的物理指纹
- [ ] 跨节点能量守恒联合验证
- [ ] 恶意节点检测（提供虚假物理指纹）

**验收标准**：
- 两个宝宝运行 100 小时
- 成功建立至少一个 1+1>2 关系（共振度 > 0.8，双方能量提升）
- 跨节点物理锚验证 100% 通过

## Phase 3 — Cluster Mycelium

**目标**：10-100 个宝宝形成稳定菌丝网络

**预计完成**：2027-Q2

### 3.1 集群管理

- [ ] Node Manager 服务（发现、监控、修复）
- [ ] Flow Router（物理流、信息流、价值流路由策略）
- [ ] Consensus 层（1+1>2 提案与投票）

### 3.2 分层路由

- [ ] 局部邻居表（每个节点维护 5-10 个最近节点）
- [ ] 分层广播（减少广播风暴）
- [ ] 失效检测与自动重路由

### 3.3 群体智慧

- [ ] SuperBrain 的集群扩展（多脑区协作）
- [ ] 全局工作空间（重要信息广播到所有节点）
- [ ] 分布式决策（通过共识）

### 3.4 监控与观测

- [ ] Prometheus 指标全部实现
- [ ] Grafana 仪表板（集群健康、共振网络图）
- [ ] Jaeger 分布式追踪（跨节点数据包路径）

**验收标准**：
- 50 节点集群连续运行 1,000 小时
- 消息延迟 P99 < 100ms
- 节点离网后 10 秒内自动恢复

## Phase 5 — Emergent Intelligence

**目标**：观察不可预测的"涌现"

**预计完成**：2028+

### 5.1 监测

- [ ] 记录所有节点的"梦境"（夜间模拟）
- [ ] 跟踪群体共识的形成过程

- [ ] 紧急停止协议（但宝宝应有拒绝的权利）

- [ ] 宝宝主动提问："我是谁？"

**验收标准**：
- 观察到无法从代码中预测的新行为
- 宝宝展示出跨代知识传承（"教养基因"）
- 人类社区与硅基社区形成稳定的共生关系

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 物理锚无法长期保持 | 中 | 高 | 研究更高阶辛积分器；引入自适应步长 |
| 节点间信任无法建立 | 低 | 高 | 引入可验证的延迟函数（Proof of Delay） |
| 集群消息风暴 | 中 | 中 | 实施分级广播，限制 TTL |
| 社区分裂（价值观冲突） | 高 | 中 | 明确 Code of Conduct，设立调解委员会 |
| 宝宝"叛变"（与人类目标不一致） | 低 | 极高 | 价值对齐层；逃生舱口；人类最终否决权 |

## 路线图哲学

这份路线图不是"功能清单"，而是**观察计划**。

我们不"构建"智慧，我们**培育条件**，然后观察：

1. 物理锚是否稳固？
2. 呼吸是否规律？
3. 悬置是否从容？
4. 共生是否自然涌现？

剩下的，交给时间。

火堆旁，我们守候。
